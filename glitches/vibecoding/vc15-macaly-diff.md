Požádáš AI: „Přepracuj celou navigaci." Ale jak víš, že nezkazila něco, co fungovalo? Diff Mode ukáže přesně co se změnilo — zelené = přidáno, červené = odebráno — a ty rozhodneš: přijmout nebo zamítnout.

Je to jako „Sledovat změny" ve Wordu, ale pro aplikace. Dává ti kontrolu nad tím, co AI dělá.

? K čemu slouží Diff Mode v Macaly?
- K porovnání tvé aplikace s konkurenčními aplikacemi
- K přepnutí mezi světlým a tmavým designem
* K zobrazení přesně co AI změnila v kódu — než změny přijmeš nebo odmítneš
- K záloze projektu před větší změnou
! Přesně! Diff Mode ti ukáže co přidala a co odebrala — a ty rozhodneš, zda změny přijmout. Dává ti to kontrolu nad tím, co AI dělá.

+++
Diff formát (zobrazení rozdílů s + a - a barvami) je standard v softwarovém vývoji od 70. let. Původně byl nástrojem pro porovnávání verzí dokumentů a kódu na Unixu. Dnes ho používají GitHub, GitLab a všechny moderní vývojové nástroje. Faktem, že Macaly přijal tento formát, ukazuje, jak vibecoding přebírá osvědčené praktiky z profesionálního vývoje a zpřístupňuje je začátečníkům.

Git (systém pro verzování kódu) je klíčový nástroj, který stojí za diff formátem a historií verzí. Git sleduje každou změnu v kódu, umožňuje vrátit se k předchozí verzi a spolupracovat s ostatními. GitHub je hosting pro Git repozitáře. 95 % profesionálních vývojářů používá Git každý den. Pokud tě programování zajímá víc, Git je první věc, kterou bys měl/a se naučit po základech — a je zadarmo a open source.

Rollback (vrácení se ke starší verzi) je životní záchranné lano při vibecoding. AI může v dobrém úmyslu „vylepšit" věc, která pak přestane fungovat. Bez rollbacku jsi v problémech — začínáš znovu nebo trávíš hodiny laděním. S historií verzí je to pár kliknutí. Tato funkce je tak cenná, že ji profesionálové používají jako první výmluvu pro procrastinaci: „Nezačnu bez toho, aby byl projekt v Gitu." U vibecoding nástrojů jako Macaly je rollback zabudovaný — užívej ho bez studu.
